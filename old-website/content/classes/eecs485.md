## Course Overview

EECS 485 is an intensive course on web application development, covering both client-side and server-side technologies. The course emphasizes building production-quality web systems with proper architecture, testing, and deployment practices.

## Key Topics

### Frontend Development
- Modern JavaScript (ES6+) and TypeScript
- React and component-based architecture
- Client-side routing and state management
- Responsive design and accessibility

### Backend Development
- RESTful API design
- Server-side rendering vs client-side rendering
- Database design and SQL
- Authentication and authorization
- Session management

### DevOps & Deployment
- Containerization with Docker
- Continuous integration/deployment
- Cloud deployment (AWS)
- Performance optimization

### Software Engineering Practices
- Version control with Git
- Test-driven development
- Code reviews and pair programming
- Agile methodologies

## Projects

### Project 1: Static Site Generator
Built a Python-based static site generator that converts markdown to HTML using Jinja2 templates. Learned about build systems and template engines.

### Project 2: Server-Side Dynamic Pages
Implemented a Flask application with server-side rendering, database integration, and user authentication. Focused on proper MVC architecture.

### Project 3: Client-Side Dynamic Pages
Created a REST API backend and React frontend for an Instagram-like application. Learned about AJAX, infinite scroll, and optimistic updates.

### Project 4: MapReduce
Implemented a distributed MapReduce framework for processing large datasets. Explored parallel computing and fault tolerance.

### Project 5: Search Engine
Built a scalable search engine with web crawling, inverted index, PageRank, and a search interface. Integrated all course concepts into one large system.

## Skills Gained

- Full-stack web development proficiency
- Understanding of distributed systems
- Ability to design and implement complex web applications
- Strong debugging and testing skills
- Experience with modern development tools and practices

## Favorite Aspects

The course projects are incredibly practical and build on each other well. Each project introduces new concepts while reinforcing previous ones. The final search engine project is particularly satisfying as it brings everything together.

## Challenges

The projects are time-intensive and require careful planning. Debugging distributed systems can be tricky. Time management is crucial, especially when balancing with other coursework.

## Takeaways

This course provides excellent preparation for software engineering roles. The emphasis on building real systems (not just toy examples) makes the learning stick. The skills learned here directly apply to personal projects and internships.
